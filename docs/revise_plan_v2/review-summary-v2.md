## 전원 공통 지적 (v3에 반드시 반영)

1. **Tool Use 2-tool → 1-tool + text** 단순화 (CEO/Eng 공통)
2. **데모 역할 전환 메커니즘** 필수 (Design 치명적 지적)
3. **학생 채팅 품질에 올인** > 기능 수 확장 (Competition 핵심 결론)
4. **Non-streaming fallback 먼저** 구현 (Eng 리스크 완화)
5. **Render cold start 방지** (Eng/CEO 공통)

## v3 핵심 원칙

> **"학생 채팅 1개 기능의 완벽함 > 기능 10개의 존재"**
> — Competition v2 리뷰어

이 원칙에 따라:
- Admin 대시보드를 P2로 강등 (학생 채팅 + 교강사 대시보드 품질에 집중)
- Iron Rule: Day 3 끝까지 학생 채팅이 라이브에서 동작해야 함
- AI 리포트를 매일 기록 (Day 7에 급조하지 않음)
