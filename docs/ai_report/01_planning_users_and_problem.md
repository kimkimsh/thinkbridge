# 1-1. 설정한 사용자는 누구이며, 해결하려는 구체적인 문제점/불편함은 무엇인가요?

## 한 줄 답변

> **중고등학생·교강사·학원 관리자 3계층을 사용자로 설정하여, "AI가 답을 주는 시대에 사라져가는 사고 훈련의 공백"을 해결한다.**

## 3계층 사용자 프로필

ThinkBridge는 학원 환경의 **세 역할 모두**를 1차 사용자로 설계했다. "학생만 쓰는 제품"이 아니라 **학습 사이클 전체**가 대상이다.

| 사용자 | 구체 프로필 | 현재 겪는 불편함 |
|--------|-------------|------------------|
| **학생** | 중·고등학생, 수학·과학·논술 수강생. 주제를 받으면 바로 ChatGPT에 복붙하여 정답을 구한다 | 답은 얻지만 **"왜 그렇게 되는가"** 의 사고 과정이 몸에 배지 않는다. 시험장에서 응용 문항이 나오면 무너진다 |
| **교강사** | 수학/과학/논술 강사. 20명 학생을 일괄 지도하지만 개별 사고 수준은 **감으로** 판단한다 | 어떤 학생이 "논리 구조화"에서 약한지, "근거 제시"에서 약한지 **데이터 없이 파악 불가**. 맞춤 지도가 사실상 불가능하다 |
| **학원 관리자** | 학원 운영 책임자. AI 시대에 인간 교사의 가치를 어떻게 차별화할지 고민한다 | 학부모 상담에서 "우리는 AI와 다르다"고 설명할 근거가 **추상적**. 데이터로 보여줄 수 없다 |

## 왜 지금 이 문제가 중요한가

2024년 이후 ChatGPT/Claude 사용이 일반화되면서, 교육계는 "AI가 과제를 대신 해주는 시대"에 진입했다. 그러나:

1. **AI는 답을 주지만 사고 능력을 길러주지 않는다.** 빠른 정답에 의존할수록 문제 해결 근육이 퇴화한다.
2. **교강사는 "AI를 쓰지 말라"고 말할 수밖에 없는 수동적 입장**이다. 금지는 지속 불가능하다.
3. **교육의 본질이 답이 아닌 사고 과정**임을 모두 알지만, **"사고 과정을 훈련시키는 도구"** 가 없다.

## 우리가 정의하는 구체적 세부 문제 3가지

### 문제 ① 학생의 "사고 스킵" 현상
- **현 상태**: 문제 → AI → 정답 복사 (3~10초)
- **목표 상태**: 문제 → AI의 질문 → 스스로 고민 (5~15분) → 본인이 답 도출

**수학 시나리오 비교**:
- *Before*: "x² − 5x + 6 = 0 풀어줘" → AI: "x = 2, 3" → 복사 → 끝
- *After*: 동일 입력 → AI: "이 식이 인수분해 가능한지 먼저 확인해볼까요?" → 학생이 스스로 풀이 → AI는 **끝까지 답을 주지 않음** → 학생이 "곱해서 6, 더해서 −5니까 −2와 −3"에 스스로 도달

### 문제 ② 교강사의 "블라인드 지도" 구조
- **현 상태**: 시험 점수만 보고 "수학이 약하다" 정도만 파악
- **목표 상태**: **6차원 사고력 프로필**(문제 이해·전제 확인·논리 구조화·근거 제시·비판적 사고·창의적 사고)로 **구체적 약점 차원 진단**

### 문제 ③ 학원의 "차별화 증명" 부재
- **현 상태**: "우리는 인성 교육도 합니다" 같은 슬로건
- **목표 상태**: 학생별 **사고력 성장 추이 그래프**를 학부모에게 제시 → "이번 달 아이의 비판적 사고 점수가 +2.3 상승"이라는 **데이터 기반 소통**

## 문제 정의의 원천 ("왜 이 방향인가")

본 문제 정의는 팀 내부 3차례 계획 수정(`docs/revise_plan_v1 → v2 → v3`)을 거치며 다듬어졌다. 특히 v2 단계의 **Competition Reviewer 에이전트**가 제시한 관점:

> "AI 시대에 '학생에게 AI를 사용시키지 말라'는 해법은 작동하지 않는다. 대신 **AI를 통해 AI와는 다른 역량(사고력)을 기르는** 제품이 필요하다."

이 통찰이 "답을 주지 않는 AI"라는 역발상 제품 정의의 씨앗이다.
